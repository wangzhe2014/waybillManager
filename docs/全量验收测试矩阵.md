# V3 全量验收测试矩阵

目标：在干净 V3 测试库上，覆盖页面主流程、错误路径、状态机限制和基础配置能力。

## 测试数据准备

1. 确认当前连接的是 V3 独立测试数据库，不是生产库。
2. 执行 `supabase/schema.sql`，重建表和函数。
3. 执行 `supabase/reset_acceptance_data.sql`，清空并写入验收数据。
4. 启动 V2 服务和 V3 服务，确认 V3 能正常访问 V2 HTTP 接口。

## 自动化检查

| 编号 | 命令 | 验收点 |
| --- | --- | --- |
| A1 | `npm run typecheck` | TypeScript 类型通过 |
| A2 | `npm test` | 核心状态机、规则、审批、执行、监控、查询服务通过 |
| A3 | `npm run test:full` | 类型检查和全部单元测试一次性通过 |

| 编号 | 分支 | 验收点 |

| D1 | 有工单、有扫描、有日志 | 指标不显示假数据或闪烁追加 |

## 回归边界

| 编号 | 分支 | 验收点 |
| --- | --- | --- |
| B1 | 刷新任意页面 | 不先显示旧本地数据再追加接口数据 |
| B2 | 连续快速切换筛选 | 不出现旧请求覆盖新请求 |
